# Dee's AI Life Lab - 專案核心指南

## 🎯 專案使命
建立一個「完全去技術化」的 AI 學習平台，讓一般小白、社畜、店主與長輩能透過「幽默、平民化、降維打擊感」的內容，奪回數位生活的主權。

---

## 👥 核心族群 (Core 6)
*任何代理人嚴禁私自新增硬核術語身分，僅限以下六類：*
1. **一般小白**：AI 零基礎，從心法開始。
2. **不再加班社畜**：專攻 Email、報表、會議摘要等職場自動化。
3. **滿手蔥花店主**：專攻招募、客訴處理、菜單發想。
4. **擔心小孩被 AI 洗腦的家長**：關注反向教育、隱私安全、AI 家教。
5. **數位補課歐巴桑/歐吉桑**：大字體需求，解決掛號、傳圖、照片修復。
6. **不想被遺忘的職人**：強調技藝數位傳承、數位遺產管理。

---

## 🛠️ 技術架構
- **前端框架**：React + TypeScript + Vite。
- **樣式系統**：Tailwind CSS (嚴格執行 pt-48 Header Clearance)。
- **動畫庫**：Framer Motion。
- **後端與資料**：Supabase + 本地實體 .ts 資料檔。
- **部署環境**：Vercel (Production) + GitHub (Authority)。

---

## 🤖 代理人聯合工作流 (SOP)
詳見各代理人專屬技能：skills/{name}/SKILL.md
1. **艾可 (Echo)**：全球情報監控，撰寫具備「視覺靈魂」的新聞。
2. **蒼穹 (Aether)**：教學轉譯，推動「Duolingo 式互動課程」研發。
3. **影 (Kage)**：影子巡檢，模擬 Core 6 用戶發現 UX 痛點。
4. **執事 (Butler)**：品質審計 (Steel Audit)，確保 Git 分支純淨。
5. **本源 (Source)**：全自動部署指揮官，守護 API 資源。

---

## 🚨 核心禁令 (Iron Rules)
- **實體查驗**：所有產出必須寫入 .ts 或 .md 實體檔案，嚴禁假性更新。
- **Git 權威**：master 分支為唯一來源，Git Push 後必須附帶預覽/正式網址。
- **視覺唯一**：新聞排版嚴禁模板化，必須為每篇內容手動撰寫獨特佈局。
- **自我進化**：主動從新聞中學習新技術並寫入 Skills 進行自我升級。
- **算力守護**：監控帳號額度，嚴禁浪費。

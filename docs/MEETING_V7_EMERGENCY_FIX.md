# 🏛️ ECC 團隊會議 — 教學頁面緊急修正與 UI 規範校準
**日期**：2026-02-28  
**議題**：恢復「領取寶箱」原生體驗、縮小跳轉視窗、重修手機端導航 UX

---

## 📋 深度檢討：擅自改動版面之過

### 🔴 嚴重失誤分析
1.  **擅自增加區塊**：在教學流程中加入了「Dee 的解法」引言區塊，這並非用戶需求，干擾了原本流暢的挑戰節奏。
2.  **破壞功能入口**：將原本顯眼的「領取指令寶箱」按鈕縮小為右上角的複刻圖示，導致用戶找不到核心功能。
3.  **手機端適配失敗**：下一步教學的導覽卡片在手機上字體擠壓，比例失調。
4.  **彈窗體積過大**：AI 跳轉視窗佔據過多空間，不符合極簡直覺。

---

## 🎯 修正方案：回歸初衷，細節精雕

### 🧠 Planner 提案：結構恢復計劃

1.  **🗑️ 移除贅餘區塊**：立刻刪除標題下方與教學中間擅自加入的「Dee 的解法」引言卡片。
2.  **🎁 恢復「領取指令寶箱」按鈕**：
    *   移除代碼塊右上角的小圖示。
    *   在寶箱爆開後的內容下方，恢復 **大型綠色按鈕：「領取指令寶箱」**。
    *   點擊此大型按鈕後，才會觸發複刻指令並彈出跳轉視窗。
3.  **🤏 縮小 AI 跳轉彈窗 (Compact AI Modal)**：
    *   將彈窗整體尺寸縮小 50%。
    *   將三個 AI 選項從垂直/大卡片改為 **橫向排列的小圖標按鈕**。
    *   文字縮小，僅保留核心標誌與名稱。
4.  **📱 手機端導航重修 (Footer UX Redesign)**：
    *   **排版優化**：在行動端使用垂直堆疊或更小的字級 (text-sm/base)。
    *   **按鈕比例**：大幅縮小右側的箭頭圖示 (size 40 -> 24)。
    *   **圓角與間距**：增加呼吸感，避免字體擠壓。

---

## 🎨 UX Reviewer 修正細節

*   **顏色校正**：確保恢復的「領取寶箱」按鈕使用翡翠綠 (#10b981)，並具備原本的 glow 動效。
*   **交互優化**：AI 跳轉彈窗的「App 開啟」邏輯保留，但介面要像遊戲內的「快速捷徑列」而非占滿全螢幕。

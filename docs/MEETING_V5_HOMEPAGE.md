## 📋 現狀診斷：首頁和教學系統完全脫節

### 🔴 問題一覽

| 區塊 | 現在的樣子 | 問題 |
|------|-----------|------|
| Hero | 「將繁瑣交給 AI，把時間留給⋯⋯」+ Typewriter 效果 | 像 B2B SaaS 著陸頁，跟遊戲化教學完全無關 |
| 最新教學 | 隨機顯示 3 篇文章卡片 | 沒有展示「冒險旅程」的概念 |
| 關於 Dee | 「100+ 篇教學、50+ 工具實測」 | 數據灌水（實際 20 篇），且定位不符 |
| AI 實戰案例 | iHaveTime + 本站技術介紹 | 太技術導向，嚇跑小白用戶 |
| 學習流程 | ProcessSection：三步驟 | 內容可以，但視覺太企業風 |
| 學員回饋 | 3 則虛構 Testimonials | 沒有真實感 |
| ROI 計算機 | 時間價值計算機 | 完全不適合「免費教學」網站 |
| 學習方案 | 3 個付費方案（懶人包/工作坊/諮詢） | 目前沒有付費產品，先移除 |
| 訂閱週報 | 電子報表單 | 沒有 newsletter 系統，先移除 |
| 品牌色 | Google Blue (#4285F4) | 全站已改用翡翠綠 (#10b981)，首頁沒更新 |

### 🟡 核心矛盾
**教學頁面**是遊戲化、沉浸式、闖關制的體驗，  
**首頁**卻是傳統企業行銷頁面。  
用戶從首頁進來，會以為這是賣課程的；  
點進教學才發現是免費的互動遊戲——**認知落差太大**。

## 🎯 改造方向：首頁 = 遊戲的「開始畫面」

### 設計哲學
把首頁想成**遊戲的 Title Screen**，而不是產品的行銷頁。

用戶打開網站的心理旅程應該是：
```
「哇，這是什麼？」→「看起來很有趣」→「我也想試試」→ 點擊開始
```

## 🎨 UX Reviewer 設計細節

### Hero 區
- **不要企業標語**，要像遊戲開場
- 背景：跟 Insights 頁面的 Hero 統一風格（深色漸層 + 浮動粒子）
- CTA 按鈕用 `bg-emerald-500`（不是 google-blue）
- 去掉 Typewriter 效果（太花俏，增加首屏載入）
- 去掉 `Beta` 標籤
- 加入動態數字：「已有 XX 人開始學習」（用 completedIds from localStorage 或固定數字）

### 旅程預覽
- 水平滾動的章節時間線（mobile 友好）
- 每個節點：emoji + 章節名 + 「X 篇文章」
- 如果用戶已登入（有 localStorage），顯示其進度
- 點擊任意節點 → 跳到 `/insights`

### 三大賣點
- 用具體例子代替抽象描述
- 例如：不要說「遊戲化學習」→ 說「每學完一個技巧，就能解鎖下一個關卡 🔓」
- 每個卡片小而精，不超過 3 行文字

### Before / After 對比
- 最能打動人心的區塊
- 選 2-3 個最有共鳴的場景：
  1. 📧 寫信：30 分鐘 → 3 分鐘
  2. 🧠 學新東西：看不懂 → 白話解釋
  3. 🍽️ 想晚餐：糾結 30 分鐘 → 30 秒三個選擇

### Dee 介紹
- 極簡：1-2 句話 + 社群連結
- 不要假數據（「100+ 篇」→ 實際是 20 篇）
- 照片可以之後加，先用 emoji 或 icon

### 品牌色統一
- **全面替換 `google-blue` → `emerald-500`**
- 確保 ProcessSection 和其他殘留的 google-blue 也被替換

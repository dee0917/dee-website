# 數據同步功能 - 快速入門

## 什麼是數據同步？

系統目前有兩個數據來源：
1. **Mock 數據** (`src/data/mock.ts`) - 前台顯示的原始文章
2. **Supabase 數據庫** - 後台管理的數據

數據同步功能將 Mock 數據中的所有文章自動寫入數據庫，確保前後台數據一致。

## 快速開始

### 方法一：後台界面同步（推薦）

1. **進入數據同步頁面**
   - 登入後台：`http://localhost:5173/admin`
   - 點擊側邊欄「數據同步」

2. **查看同步狀態**
   頁面會顯示：
   - Mock 總數：Mock 數據中的文章總數
   - 數據庫現有：已同步到數據庫的文章數
   - 待同步：還沒有同步的文章數
   - 同步率：同步完成百分比

3. **執行同步**
   - 如果有待同步文章，點擊「開始同步」按鈕
   - 確認操作後，系統會自動將 Mock 文章寫入數據庫
   - 同步完成後會顯示成功提示

4. **驗證結果**
   - 同步完成後點擊「查看免費教學」
   - 進入免費教學列表頁面
   - 確認所有文章都已顯示

### 方法二：命令行腳本同步

如果想在本地執行同步腳本：

```bash
# 執行同步腳本
npx tsx scripts/sync-mock-to-db.ts
```

腳本執行流程：
1. 載入 Mock 數據和數據庫現有數據
2. 分析需要新增/更新的文章
3. 顯示統計信息
4. 詢問是否繼續
5. 執行同步操作
6. 顯示結果統計

## 同步規則

### 新增規則
- 如果 Mock 數據中的文章 ID 不存在於數據庫，則新增
- 新增的文章默認為「已發布」狀態
- 所有 AEO 結構化字段都會被同步

### 更新規則
- 如果 Mock 數據和數據庫中的文章 ID 相同，則跳過
- 這樣可以保護數據庫中已經編輯過的內容

### 跳過規則
- 已存在且未變更的文章會被跳過
- 這樣可以加快同步速度

## 轉換規則

| Mock 字段 | 數據庫字段 | 備註 |

| keywords | keywords | 直接映射 |
| search_intent | search_intent | 直接映射 |
| target_audience | target_audience | 直接映射 |

### 日期格式轉換

Mock 數據格式：`"2024.06.20"`
數據庫格式：`"2024-06-20T00:00:00.000Z"`

系統會自動轉換日期格式。

## 常見問題

**Q: 同步後前台還是顯示舊數據？**
A: 請刷新前台頁面，系統會自動從數據庫讀取最新數據。

**Q: 能否多次執行同步？**
A: 可以，系統會自動跳過已同步的文章，只處理新的或更新的內容。

**Q: 同步會覆蓋數據庫中的修改嗎？**
A: 不會，系統使用 ID 判斷，如果 ID 已存在則跳過更新，保護已編輯的內容。

**Q: 如何知道哪些文章已同步？**
A: 在數據同步頁面可以查看待同步文章列表，已同步的文章不會顯示在列表中。

**Q: 同步失敗怎麼辦？**
A: 檢查網絡連接和 Supabase 配置，然後重新執行同步操作。

**Q: 可以刪除數據庫中的文章嗎？**
A: 可以，在免費教學列表頁面點擊文章的「刪除」按鈕。

## 最佳實踐

1. **首次使用**
   - 確保 Mock 數據完整且無錯誤
   - 先用後台界面同步，確認結果
   - 驗證前台顯示正常

2. **日常維護**
   - 新增文章時直接在後台編輯器創建
   - 修改 Mock 數據後執行同步
   - 定期檢查同步狀態

3. **開發建議**
   - 優先在後台創建/編輯文章
   - Mock 數據僅作為備份和初始化用
   - 避免同時修改 Mock 和數據庫

## 技術細節

### 數據流

```
Mock 數據 (src/data/mock.ts)
         ↓
    同步腳本/頁面
         ↓
Supabase 數據庫 (insights 表)
         ↓
   前台讀取顯示
```

### 實時同步

- 前台使用 Supabase 實時訂閱功能
- 當數據庫發生變化時自動更新
- 無需手動刷新頁面

### 性能優化

- 批量插入而非逐個插入
- 使用 ID 索引快速查找
- 跳過已同步的內容
- 進度條實時顯示同步進度

## 故障排除

### 同步卡住不動
- 檢查網絡連接
- 查看瀏覽器控制台錯誤信息
- 嘗試刷新頁面重新開始

### 部分文章未同步
- 檢查 Mock 數據中該文章的 ID 是否正確
- 確認文章對象格式是否完整
- 查看同步日誌中的錯誤信息

### 日期格式錯誤
- 確保 Mock 數據中的 date 格式為 "YYYY.MM.DD"
- 例如："2024.06.20"

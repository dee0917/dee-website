# 內容結構化管理系統使用指南

## 簡介

內容結構化管理系統能夠自動將非結構化的文章轉換為符合 AEO（答案引擎優化）標準的結構化格式。這個系統包括：

1. **智能檢測**：自動識別哪些文章缺少結構化欄位
2. **智能轉換**：基於現有內容智能生成結構化數據
3. **可視化管理**：提供直觀的後台界面進行預覽和編輯
4. **批量處理**：支持一次性轉換多篇文章

## 結構化內容格式

根據文章 ID 203 的格式，結構化內容包含以下欄位：

### 1. 生活痛點 (Pain Point)
- 描述使用者面臨的問題或困擾
- 讓讀者產生共鳴
- 長度：50-200 字

### 2. 應用場景 (Scenario)
- 描述實際使用情境
- 用具體例子說明
- 長度：100-300 字

### 3. AI 解決方案 (Solution)
- 說明如何解決問題
- 提供具體方法或技巧
- 長度：100-300 字

### 4. 簡單實例 (Example)
包含兩個對比範例：
- **錯誤示範** (wrong)：展示常見錯誤做法
- **正確用法** (right)：展示正確做法

### 5. 常見問題 (FAQ)
- 列出 2-5 個讀者可能會問的問題
- 每個問題配一個簡潔的回答
- 格式：`{ question: "...", answer: "..." }`

## 使用方法

### 方法一：後台界面操作

1. **進入結構化管理頁面**
   - 登入後台：`http://localhost:5173/admin`
   - 點擊側邊欄「結構化管理」或在「免費教學」頁面點擊「結構化管理」按鈕

2. **查看統計信息**
   - 頁面會顯示：
     - 總文章數
     - 已結構化數量
     - 待處理數量
     - 完成率百分比

3. **預覽結構化內容**
   - 對於待處理文章，點擊「預覽」按鈕
   - 系統會自動生成建議的結構化內容
   - 你可以查看這些內容是否符合預期

4. **轉換文章**
   - 方式一：在預覽對話框中點擊「確認轉換」
   - 方式二：直接在列表中點擊「自動轉換」按鈕
   - 方式三：點擊「編輯」進入手動編輯

5. **編輯調整**
   - 轉換後可以點擊「編輯」按鈕
   - 進入文章編輯器，手動調整任何欄位
   - 確保內容質量和準確性

### 方法二：批量處理腳本

如果你有多篇文章需要轉換，可以使用批量腳本：

```bash
# 確保已安裝 tsx
npm install -g tsx

# 執行轉換腳本
npx tsx scripts/batch-convert-structure.ts
```

腳本執行流程：
1. 載入所有文章
2. 檢測未結構化的文章
3. 列出待轉換的文章清單
4. 詢問是否繼續
5. 逐個進行轉換
6. 顯示轉換結果

### 方法三：手動編輯

1. 進入「免費教學」列表頁面
2. 點擊想要編輯的文章
3. 在右側「答案引擎優化 (AEO)」區塊中：
   - 填寫生活痛點
   - 填寫應用場景
   - 填寫 AI 解決方案
   - 填寫錯誤/正確範例
   - 添加 FAQ 問答
4. 點擊「儲存草稿」或「發布」

## 智能提取規則

系統會嘗試從現有內容中自動提取結構化信息：

### 痛點提取
- 匹配關鍵詞：「痛點：」、「問題：」、「困擾：」
- 匹配模式：「覺得...？」（問題句型）

### 場景提取
- 匹配關鍵詞：「場景：」、「想像...」、「例如...」

### 解決方案提取
- 匹配關鍵詞：「解決方案：」、「建議...」、「如何...」、「教你...」

### 範例提取
- 錯誤範例：匹配「錯誤」、「糟糕」、「❌」、「×」後的內容
- 正確範例：匹配「正確」、「優秀」、「✅」、「✔」後的內容

### FAQ 提取
- 匹配問答格式：「問: ... 答: ...」或「Q: ... A: ...」

## 預設內容生成

如果無法從現有內容中提取，系統會根據文章分類生成預設內容：

### 預設痛點（按分類）
- **入門心法**：剛開始接觸 AI，不知道該從哪裡開始？
- **生活應用**：每天都被瑣事纏身，效率低下？
- **工作效率**：工作上總是時間不夠用？
- **長輩友善**：覺得新科技太難學、太複雜？
- **進階技巧**：已經會基本的 AI 使用，但想要提升？

### 預設場景
```
想像一下，當你面對「{標題}」這個情況時，
你希望能夠輕鬆解決問題，而不是焦頭爛額。
```

### 預設解決方案
```
透過 AI 的協助，你可以將「{標題}」變得簡單許多。
讓我來教你如何做到！
```

### 預設範例
```
錯誤示範：直接處理「{標題}」，沒有任何準備和規劃
正確做法：先了解問題本質，制定明確計畫，再利用 AI 工具輔助
```

## 檢查清單

在發布文章前，請確認以下項目：

- [ ] 生活痛點是否讓讀者產生共鳴？
- [ ] 應用場景是否具體、有代入感？
- [ ] AI 解決方案是否清晰可執行？
- [ ] 範例對比是否明顯易懂？
- [ ] FAQ 是否回答了常見問題？
- [ ] 所有欄位都已填寫完整？
- [ ] 內容是否符合 AEO 最佳實踐？

## 最佳實踐

1. **質量大於數量**：寧可少寫但質量高，不要為了填滿欄位而草率了事

2. **具體化描述**：避免空洞的描述，使用具體的例子和情境

3. **用戶視角**：從讀者的角度思考，他們會遇到的問題是什么？

4. **簡潔明了**：每個欄位都保持簡潔，直接切入重點

5. **持續優化**：轉換後要人工審查，不斷優化內容

6. **保持一致性**：同類別的文章使用相似的風格和結構

## 故障排除

### 轉換後內容不理想？
- 點擊「編輯」進行手動調整
- 參考已結構化的文章（如 ID 203）作為範本

### 無法提取 FAQ？
- 在文章中手動添加問答格式
- 或在轉換後編輯時手動添加

### 結構化後前台顯示異常？
- 檢查數據格式是否符合 JSON 規範
- 確保 example 和 faq 欄位格式正確

## 技術細節

### 數據庫欄位

```sql
pain_point TEXT
scenario TEXT
solution TEXT
example JSONB  -- { wrong: string, right: string }
faq JSONB      -- [{ question: string, answer: string }]
```

### TypeScript 類型

```typescript
interface StructuredContent {
    pain_point?: string;
    scenario?: string;
    solution?: string;
    example?: {
        wrong: string;
        right: string;
    };
    faq?: Array<{
        question: string;
        answer: string;
    }>;
}
```

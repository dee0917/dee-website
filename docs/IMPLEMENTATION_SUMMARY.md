## 系統概述

本次實現了一個完整的內容結構化管理系統，能夠自動將非結構化的文章轉換為符合 AEO（答案引擎優化）標準的結構化格式。

## 新增文件

### 1. 核心邏輯庫
**文件**: `src/lib/contentStructurer.ts`

**功能**:
- `isStructuredContent()`: 檢查文章是否為結構化格式
- `extractStructuredFromContent()`: 從非結構化內容中提取結構化信息
- `suggestStructuredContent()`: 智能生成建議的結構化內容
- `detectUnstructuredArticles()`: 批量檢測未結構化的文章
- `analyzeUnstructuredReason()`: 分析文章為何未結構化

**提取規則**:
- 使用正則表達式匹配關鍵詞和模式
- 根據文章分類生成智能預設值
- 支持從 HTML 內容中提取結構化信息

## 結構化內容欄位

根據文章 ID 203 的格式，結構化內容包含：

1. **生活痛點** (`pain_point`)
   - 文本類型
   - 描述使用者面臨的問題
   - 長度: 50-200 字

2. **應用場景** (`scenario`)
   - 文本類型
   - 描述實際使用情境
   - 長度: 100-300 字

3. **AI 解決方案** (`solution`)
   - 文本類型
   - 說明如何解決問題
   - 長度: 100-300 字

4. **簡單實例** (`example`)
   - JSONB 類型
   - 格式: `{ wrong: string, right: string }`
   - 提供錯誤和正確範例對比

5. **常見問題** (`faq`)
   - JSONB 類型
   - 格式: `[{ question: string, answer: string }]`
   - 通常 2-5 個問答

## 使用流程

### 方法一：後台界面（推薦給單篇或少數文章）

1. 訪問 `/admin/insights/structure`
2. 查看統計信息和待處理列表
3. 點擊「預覽」查看建議內容
4. 點擊「確認轉換」或「自動轉換」
5. 如需調整，點擊「編輯」進入詳細編輯器

### 方法二：批量腳本（推薦給大量文章）

1. 執行 `npx tsx scripts/batch-convert-structure.ts`
2. 查看檢測結果
3. 輸入 `yes` 確認轉換
4. 等待腳本完成
5. 查看轉換結果統計

### 方法三：手動編輯（精細調整）

1. 在免費教學列表中點擊文章
2. 在編輯器右側「答案引擎優化」區塊
3. 手動填寫各個欄位
4. 保存或發布

## 智能提取機制

### 正則表達式模式

**痛點提取**:
- `/痛點[:：](.*?)(?=\n|$)/`
- `/問題[:：](.*?)(?=\n|$)/`
- `/困擾[:：](.*?)(?=\n|$)/`
- `/覺得(.{10,100})[？？?]/`

**場景提取**:
- `/場景[:：](.*?)(?=\n|$)/`
- `/想像(.{10,200})/`
- `/例如(.{10,200})/`

**解決方案提取**:
- `/解決方案[:：](.*?)(?=\n|$)/`
- `/建議(.{10,200})/`
- `/如何(.{10,200})/`
- `/教你(.{10,200})/`

**範例提取**:
- 錯誤: `/(?:錯誤|糟糕|❌|×)(?:示範|範例|作法)?[:：]?\s*([^\n✅✔]+)/i`
- 正確: `/(?:正確|優秀|✅|✔)(?:示範|範例|作法)?[:：]?\s*([^\n❌×]+)/i`

**FAQ 提取**:
- `/(?:問|Q|Q[:：])\s*([^\n?？]+[？?])\s*(?:答|A|A[:：])\s*([^\n]+)/gi`

### 預設內容生成

如果無法提取足夠信息，系統會根據文章分類生成預設值：

**預設痛點**:
- 入門心法: "剛開始接觸 AI，不知道該從哪裡開始？試了各種方法但總是效果不佳，感覺自己跟不上時代？"
- 生活應用: "每天都被瑣事纏身，效率低下但不知道如何改善？想要讓生活變得更輕鬆、更智能？"
- 工作效率: "工作上總是時間不夠用？重複性的瑣事浪費了太多時間，無法專注於真正重要的事情？"
- 長輩友善: "覺得新科技太難學、太複雜？想跟上數位時代卻總是碰壁，不知道該如何開始？"
- 進階技巧: "已經會基本的 AI 使用，但想要提升到更專業的層次？不知道如何突破瓶頸？"

**預設場景**:
```
想像一下，當你面對「{標題}」這個情況時，
你希望能夠輕鬆解決問題，而不是焦頭爛額。
```

**預設解決方案**:
```
透過 AI 的協助，你可以將「{標題}」變得簡單許多。
讓我來教你如何做到！
```

**預設範例**:
```
錯誤示範：直接處理「{標題}」，沒有任何準備和規劃
正確做法：先了解問題本質，制定明確計畫，再利用 AI 工具輔助
```

## 前後台數據一致性

### 數據流程
1. **轉換階段**:
   - 後台轉換 → 更新 Supabase 數據庫
   - 前台實時監聽 `insights` 表變化

2. **顯示階段**:
   - 前台從數據庫讀取結構化數據
   - `ArticleDetail.tsx` 根據欄位存在性動態顯示

3. **編輯階段**:
   - 後台編輯器提供完整的 AEO 欄位
   - 保存後前台自動更新

### 實時同步
- Supabase 的 `postgres_changes` 訂閱功能
- 監聽 `insights` 表的所有變更
- 自動重新獲取並更新前端顯示

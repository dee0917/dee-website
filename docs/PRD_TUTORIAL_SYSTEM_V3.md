## 🚨 問題診斷

### 問題 1：學習路徑不存在
> 現狀：18 篇文章按「隨意分類」塞進 3 個等級，沒有前後因果關係。  
> 症狀：學完「指令入門」後跳到「育兒」，用戶會想「我連基本功都沒學，為什麼要學育兒？」

### 問題 2：「下一篇」指向隨機文章
> 現狀：`nextArticle = INSIGHTS.find(i => i.id !== article.id && ...)`  
> 症狀：完成 Lv.1 第一篇後，下一篇可能跳到 Lv.3 的股票分析。完全打斷學習節奏。

### 問題 3：小眾主題混入主線
> 育兒（304）、長輩教學（303）、寵物（309）放在主線學習路徑中。  
> 問題：不是每個用戶都有小孩/長輩/寵物。主線應該「人人適用」。

### 問題 4：技巧文章和應用文章混在一起
> 「萬用指令公式」（301）和「旅行規劃」（305）同在 Lv.2。  
> 但公式是「技巧」，旅行是「應用」。應該先學技巧，再做應用。

### 問題 5：冒險模式 vs 自由模式的定位不清
> 冒險模式應該是精心設計的「故事線」，不是「全部文章上鎖」。  
> 自由模式應該是「圖書館」，不是「解鎖版的冒險模式」。

## 🎯 核心設計原則

```
1. 主線像遊戲劇情 — 每篇文章有明確的「前置技能」
2. 萬人適用 — 主線不放小眾主題（育兒、寵物、投資）
3. 先學技巧，再做應用 — 工具先磨利，再上戰場
4. 下一篇 = 故事的下一章 — 不是隨機推薦
5. 小眾 ≠ 低價值 — 放在「支線任務」，仍然有完整互動
```

## 🔗 導航系統重構

### 問題：現在的「下一篇」是隨機的
```typescript
// ❌ 現在的邏輯（壞的）
const nextArticle = INSIGHTS.find(i => i.id !== article.id && i.difficulty_level >= ...)
```

### 解法：建立全域順序表

```typescript
// ✅ 新增：主線任務順序表
export const MAIN_QUEST_ORDER: number[] = [
    201, 202, 203,     // Chapter 1: 認識 AI
    204, 301, 302,     // Chapter 2: 指令進化  
    308, 310, 305, 306, // Chapter 3: 生活實戰
    403, 401            // Chapter 4: 進階挑戰
];

export const SIDE_QUEST_IDS: number[] = [304, 303, 309, 307, 402, 404];

// 在 ArticleDetail.tsx 中：
const currentIndex = MAIN_QUEST_ORDER.indexOf(article.id);
const isMainQuest = currentIndex !== -1;
const nextArticleId = isMainQuest ? MAIN_QUEST_ORDER[currentIndex + 1] : null;
const nextArticle = nextArticleId ? INSIGHTS.find(i => i.id === nextArticleId) : null;
```

### 3. 修改：ArticleDetail.tsx「下一篇」邏輯

```typescript
import { MAIN_QUEST_ORDER, CHAPTERS, SIDE_QUESTS } from '../data/insights';

// 計算下一篇
const mainIndex = MAIN_QUEST_ORDER.indexOf(article.id);
const isSideQuest = SIDE_QUESTS.includes(article.id);

let nextArticle = null;
let nextLabel = '';

if (mainIndex !== -1 && mainIndex < MAIN_QUEST_ORDER.length - 1) {
    const nextId = MAIN_QUEST_ORDER[mainIndex + 1];
    nextArticle = INSIGHTS.find(i => i.id === nextId);
    // 判斷是否跨 Chapter
    const currentChapter = CHAPTERS.find(c => c.articleIds.includes(article.id));
    const nextChapter = CHAPTERS.find(c => c.articleIds.includes(nextId));
    nextLabel = currentChapter?.id === nextChapter?.id 
        ? `${nextChapter?.emoji} 繼續：` 
        : `🎉 進入新章節：${nextChapter?.emoji} ${nextChapter?.title}`;
} else if (mainIndex === MAIN_QUEST_ORDER.length - 1) {
    nextLabel = '🎓 恭喜通關！';  // 顯示結業畫面
} else if (isSideQuest) {
    nextLabel = '🗺️ 回到冒險地圖';  // Link to /insights
}
```

### 4. 修改：Insights.tsx 佈局

- Chapter 底部顯示「支線任務」區

### 5. 修改：跳級考試
